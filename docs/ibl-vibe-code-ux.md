# IBL Vibe Code — UX & UI Standards

Este documento define os padrões estéticos e de interação para o IBL Research Framework, garantindo uma experiência premium e academicamente rigorosa.

## 1. Tipografia e Alinhamento
- **Justificação**: Todo o conteúdo textual dentro de cartões e componentes de explicação deve ser **justificado** (`text-align: justify`).
- **Hifenização**: Utilizar `hyphens: auto` para evitar "rios" de espaço em branco no texto justificado.
- **Hierarquia**: Títulos em `font-display`, corpo em `font-body` (sans-serif moderda).

## 2. Tooltips e Micro-Interações
- **Posicionamento**: Tooltips devem aparecer sempre **acima** do componente ativador (`placement: top`).
- **Contraste**: Utilizar um tema de alto contraste (ex: fundo `var(--on_surface)` e texto `var(--surface_container_lowest)`).
- **Feedback**: Adicionar um ligeiro "scale-up" ao abrir (ex: `scale(1.02)`).

## 3. Cores e Acessibilidade
- **Paleta**: Manter a paleta "The Digital Curator" com azuis profundos (`--primary`) e realces em vermelho acadêmico (`--secondary`).
- **Contraste Mínimo**: Garantir rácio de contraste de 4.5:1 para texto normal e 3:1 para texto grande.
- **Glassmorphism**: Utilizar painéis de vidro (`glass-panel`) em modais e popups ética para profundidade visual.

## 4. Componentes Premium
- **Ethical Tips**: Devem ter um ícone 🛡️ visível e uma moldura que denote importância sem ser alarmista.
- **Progress Trackers**: Devem ser fluidos e mostrar a "thread" do conhecimento a ser construída.
